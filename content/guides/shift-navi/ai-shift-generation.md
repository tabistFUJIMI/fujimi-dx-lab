## このガイドを読むとこうなります

| Before | After |
|---|---|
| AI草案に赤字で要修正が10件以上つく | 草案のまま確定できる日が8割に |
| 生成後、毎回1時間手動調整 | 調整時間 5分以内 |
| 「なぜその配置にしたか」が分からず不信感 | 配置理由をAIが説明、納得感UP |

## AI自動生成が苦手なケース

まずは「AIが苦戦しやすいパターン」を把握すると対策が打てます:

- **希望提出ギリギリのスタッフ**: 選択肢が狭まり最適配置が困難
- **特殊資格持ちが1人のみ**: その人がいない時間帯が作れない
- **営業時間が不規則**: 曜日・時期ごとに変動する店舗

:::info
この3つに当てはまる店舗は、AI生成 → 手動調整のハイブリッド運用が前提になります。それでも8割は自動化できます。
:::

## 精度を上げる設定

### 1. 必要人数の精緻化

:::tip
**ピーク時・閑散時の差** を細かく設定するほどAIの精度が上がります。
:::

悪い例:

```
平日 11:00-23:00: キッチン2, ホール2（12時間ずっと同じ）
```

良い例:

```
平日 11:00-14:00 ランチ: キッチン3, ホール3
平日 14:00-17:00 中休み: キッチン1, ホール1
平日 17:00-20:00 通常ディナー: キッチン2, ホール2
平日 20:00-23:00 ピーク: キッチン3, ホール3
```

### 2. スタッフ制約の明記

| 制約 | どこで設定 | 効果 |
|---|---|---|
| 最大勤務時間/週 | スタッフ個別画面 | 扶養内を自動遵守 |
| 連続勤務上限 | スタッフ個別画面 | 労務トラブル防止 |
| 希望休の必ず取れる日数 | 希望収集設定 | 従業員満足度UP |
| 固定シフト | スタッフ個別画面 | 変更できない勤務を固定 |

### 3. 優先度の調整

AI生成時の「優先ルール」を調整できます:

| 優先ルール | 向いている業態 |
|---|---|
| 希望優先 | スタッフ定着率を最重視 |
| 公平配分 | アルバイト主体の店舗 |
| 人件費最適化 | 繁閑差が大きい店舗 |
| 経験者分散 | 新人とベテランをバランス配置 |

管理画面 → 「シフト → AI設定」から変更可能。

## 生成後のチェックリスト

草案が出たら以下を3分でチェック:

- [ ] 赤字マーク（必要人数不足）がある日がないか
- [ ] 同じ人だけが毎週同じ勤務になっていないか
- [ ] 希望休が全員取れているか
- [ ] 新人とベテランが同じ時間帯にバランス配置されているか
- [ ] 特殊業務（開店・閉店）の担当者がいるか

:::warning
AI は完璧ではありません。**必ず人の目で最終確認**してから確定してください。特に初回〜3回目の生成は、AIが貴店の特徴を学習中なので調整が多めです。
:::

## トラブル対処

| 症状 | 原因 | 対処 |
|---|---|---|
| 「必要人数を満たせません」エラー | 提出希望の合計が必要人数より少ない | 希望追加募集、または外部スタッフ投入 |
| 特定スタッフの勤務が偏る | 最大時間が他スタッフより大きい | 全スタッフの最大時間を見直す |
| 休みの曜日が揃わない | 希望が土日に集中 | 「公平配分」ルールに切り替え |
| 生成が5分以上終わらない | スタッフ数 × 日数 が多すぎ | 半月ごとに分けて生成 |

---

## 運用で効く小技

- **毎月1日に次月の希望収集開始** → スタッフが計画しやすい
- **希望未提出は「出勤可能」として扱う** よう設定 → 漏れても運用停止しない
- **確定後に急な変更**が出たら「個別差し替え」→ AI再計算で他の調整も自動
- **月末に実績と草案の差分**をCSV出力して振り返り → 次月のAI設定改善に使う
